### Exercise 10: Digital Biomarker Analytics & Scientific Presentation

**Type:** Data Analytics & Clinical Research Strategy 
* 

**Language/Tools:** Python (Data Analytics), Garmin Connect API (JSON Data Export)
*

**Description:** Developing a clinical scientific question centered around digital biomarkers, gathering custom observational data, and structuring an analytical pipeline to evaluate the hypothesis.

* **Background & Device Modification (Custom Approach):**
* While the exercise originally suggested using the Android Sensor Data Collection App, a review of smartphone sensor constraints showed that mobile phones lack continuous, 24/7 longitudinal data integrity since they are frequently set down or misplaced.

* To capture high-fidelity, uninterrupted health data, a **Garmin Forerunner 965** sports watch was substituted as the primary data collection tool. Custom personal health metrics were exported into native JSON files to accurately simulate and drive the analytical processing steps.

* **Core Deliverables Completed:**
  
**Hypothesis Formulation:** Defined a distinct scientific question focusing on clinical progression or therapeutic side-effects (modeled after digital biomarker benchmarks like tracking medication impact on physical balance or Parkinson's progression).

**Data Pipeline Exploration:** Modeled the exact data requirements and interpretation frameworks necessary to turn raw kinetic/biometric metrics into clinically meaningful answers.

**Scientific Presentation:** Prepared a core 3-slide presentation detailing:

1. The core clinical scientific question
2. The target sensor arrays utilized to collect the underlying data points
3. The architectural setup and results of the statistical/analytical computing environment (Python)
